# PRD: AgentBench — Agent 能力评测基准框架

## 1. 项目概述

一个可扩展的 Agent 能力评测框架，支持多维度评测、多框架接入、自动评分。用户通过 YAML 定义评测任务，框架自动执行被测 Agent、收集执行轨迹、根据评分标准打分并生成报告。

## 2. 目标用户

- **Agent 开发者**：评测自己开发的 Agent，发现短板，指导优化
- **研究者**：对比不同 Agent 框架/方案的能力差异
- **面试官**：候选人可展示对 Agent 核心概念的深度理解

## 3. 核心功能（必须做）

### 3.1 评测维度体系
- [ ] 6 大评测维度：工具使用、多步推理、记忆、指令遵循、执行效率、安全性
- [ ] 每个维度下设 2-4 个子维度，共约 18 个子维度
- [ ] 每个子维度至少 3 个评测任务（easy/medium/hard）

### 3.2 任务定义
- [ ] YAML 格式定义评测任务
- [ ] 支持定义可用工具（名称、描述、参数 Schema）
- [ ] 支持定义预期工具调用序列
- [ ] 支持 Mock API（拦截 Agent 的工具调用，返回预设结果）

### 3.3 Agent 接入
- [ ] 适配器模式对接不同 Agent 框架
- [ ] 内置适配器：Raw API（直接调 OpenAI API）、LangChain
- [ ] 适配器只需实现 `run_task` 方法即可接入

### 3.4 评测执行
- [ ] 执行器编排任务运行流程
- [ ] 超时控制（单任务默认 60s）
- [ ] 最大步数限制（防死循环，默认 10 步）
- [ ] 串行执行（MVP 阶段不做并行）

### 3.5 评分引擎
- [ ] 基于 Rubric 的规则评分（非 LLM 评分，保证可复现）
- [ ] 支持多评分项加权汇总
- [ ] 支持按维度汇总分数

### 3.6 报告输出
- [ ] 终端表格输出（MVP）
- [ ] JSON 格式导出（MVP）
- [ ] HTML 可视化报告（可选，非 MVP）

## 4. 不做（边界约束）

- ❌ 不做 Agent 训练/微调
- ❌ 不做 Web UI（CLI 优先）
- ❌ 不做分布式评测（单机优先）
- ❌ 不做实时评测（离线批量）
- ❌ 不做 LLM-as-Judge 评分（规则评分优先，保证可复现）
- ❌ 不做 Agent 框架本身的开发
- ❌ 不做与评测无关的功能（如日志平台、监控告警）

## 5. 成功指标

| 指标 | 目标值 |
|------|--------|
| 可接入的 Agent 框架数 | ≥ 2（Raw API + LangChain） |
| 内置评测任务数 | ≥ 20 |
| 评测结果可复现率 | > 90%（同一 Agent 同一任务，3 次结果一致） |
| 单次完整评测耗时 | < 5 分钟（20 个任务） |
| 代码测试覆盖率 | > 80% |

## 6. 非功能性需求

- Python 3.10+
- 核心依赖 ≤ 5 个（pyyaml, pydantic, openai, langchain, rich）
- CLI 入口，使用 `click` 或 `typer`
- 类型注解覆盖所有公开接口
- 所有配置文件有 JSON Schema 校验

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| Agent 执行不可控（死循环/幻觉） | 评测卡住 | 超时 + 步数上限 + Sandbox 隔离 |
| 评分规则难以覆盖所有情况 | 评分不准 | 先覆盖 80% 场景，边界 case 人工复核 |
| 不同 Agent 框架差异大 | 适配成本高 | 适配器基类抽象核心接口，差异下沉到子类 |
